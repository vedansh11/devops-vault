# DevOps Learning Vault

This repository contains my structured notes and hands-on learning for becoming a DevOps engineer.

The goal of this vault is to build:
- Linux fundamentals
- Git & GitHub
- DevOps practices
- Notes using Markdown + Obsidian

---

## Repository Structure

```
Devops-Vault/
├── 00-Daily/        # Daily learning logs
├── 01-Commands/     # Linux command reference (one command per file)
├── 02-Linux/        # Linux concepts and mental models
├── 03-Git/
│   ├── commands/    # Git command reference
│   ├── concepts/    # Git concepts (workflow, rebase vs merge, CI/CD)
├── Cheatsheet.md    # High-signal DevOps cheat sheet
```

---

## How to Use This Repository

- Daily learning is tracked in `00-Daily`
- Commands are documented individually for fast recall
- Concepts explain *why* and *when*, not just syntax
- Cheatsheet is used for quick reference during practice and revision
